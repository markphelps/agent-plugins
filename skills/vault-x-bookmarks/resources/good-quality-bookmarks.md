# Good Quality X Bookmark Examples

Use these examples to calibrate LLM judgment during `vault-x-bookmarks` prune
mode. A bookmark can be worth keeping even when it is short, but it needs to
preserve durable evidence, a concrete tactic, or a meaningful reference.

## Keep: Full X Article Captured

Pattern:

- Post body may be only a link.
- The source record includes a substantial `## Article` section.
- The article contains enough body text to support later synthesis.

Example judgment:

Keep `How to Build Your AI Second Brain Using Obsidian + Claude Code` because
the article explains the problem, architecture, and workflow for AI memory in
Obsidian. The source can support a future note on agent memory systems.

## Keep: Fetched Linked Content Has Substance

Pattern:

- Post points to a GitHub repo, skill page, product page, or article.
- The source record includes `## Linked Content` with meaningful text.
- The linked content identifies what the tool does, who it is for, and how it is
  used.

Example judgment:

Keep an ASO skills GitHub capture when the linked content describes keyword
research, metadata optimization, competitor analysis, and app growth. Even if
the post itself is short, the linked page is useful evidence.

## Keep: Standalone Post Contains the Actual Framework

Pattern:

- No article or fetched page is required because the post itself contains the
  substance.
- The post includes concrete steps, a prompt, a framework, a checklist, or a
  complete tactical observation.

Example judgment:

Keep a market-research prompt post when the prompt text is included in the
record. The source is directly reusable and can support future product-research
work.

## Keep: Thin But Clearly Important Reference

Pattern:

- The source is short, but it points to an important person, project, market, or
  competitor already relevant to the vault.
- Deleting it would remove a useful breadcrumb for a known concept or project.

Example judgment:

Keep a Karpathy LLM wiki source when it links to the canonical gist and explains
the idea-file concept, because the vault already tracks agent-first knowledge
systems.

## Keep: App Marketing Evidence With Specific Claims

Pattern:

- The record preserves numbers, channels, pricing, revenue claims, or concrete
  go-to-market mechanics.
- The claims could later become evidence in an app-marketing synthesis.

Example judgment:

Keep a post or article about growing apps to revenue when it includes the
channel, offer, funnel, pricing, or operating system behind the growth.
